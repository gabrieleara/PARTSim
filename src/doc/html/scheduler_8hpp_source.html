133  class TaskModelCmp {
134  public:
135  /*
136  Remember that lower numbers mean higher priorities...
137 
138  This function returns true if "a" has higher priority
139  than "b".
140 
141  - when 2 tasks have the same priority (i.e. deadline in
142  EDF*), then the priority is decided basing upon the
143  insertion time (see setInsertTime() method).
144 
145  - when 2 tasks have the same priority and the same
146  insertion time, then the priority is decided based
147  upon the task number (lower number => higher priority)
148 
149  It is possible to specify a "slice" time for each
150  task. This slice time models the quantum in round robin
151  schedulers. When a task starts executing, it is
152  assigned a timeout equal to the slice size. If the
153  timeout expires while the task is still executing, the
154  task is suspended, removed from the ready queue, and
155  re-inserted with a higher insertion time. In this way,
156  it is like inserting back in the queue of tasks with
157  the same priority.
158 
159  By using the slice time, it is possible to implement
160  the SCHED_RR policy of POSIX, by using a fixed
161  priority, and setting the slice time for each task.
162  */
163  bool operator()(TaskModel* a, TaskModel* b) const;
164  };
165  };
